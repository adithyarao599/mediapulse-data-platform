# Security Policy

## Supported Versions

MediaPulse Data Platform is currently maintained as a portfolio and learning project.

The latest version is considered the supported version for documentation updates and future enhancements.

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅         |
| < 1.0   | ❌         |

---

# Reporting a Security Issue

If you discover a security-related issue, please avoid creating a public issue that exposes sensitive information.

Instead, provide the following details:

* Description of the issue
* Steps to reproduce
* Potential impact
* Suggested remediation (if available)

As this is a public portfolio project, reported issues will be reviewed and addressed as time permits.

---

# Security Best Practices

MediaPulse follows several security-oriented development practices:

* Environment variables for configuration
* Separation of configuration from application code
* Modular architecture
* Dependency management through Python packages
* Version control with Git
* Docker-based isolated development environment

---

# Future Security Enhancements

Planned improvements include:

* Secret management
* Dependency vulnerability scanning
* Container image scanning
* Automated security checks in CI/CD
* Cloud IAM integration
* Audit logging enhancements

# Disclaimer

MediaPulse is intended as a demonstration of enterprise data engineering concepts and should not be considered production-ready software without additional security hardening, operational monitoring, and infrastructure controls.
